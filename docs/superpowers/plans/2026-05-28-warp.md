# Warp kernel transform + torus_knot Lua sample

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `warp` kernel transform whose deformation function is a Lua code string carried in `params.code`, plus a `torus_knot` Lua showcase that uses it.

**Architecture:**
The kernel's evaluate path becomes async because `warp` must call into a Lua runtime per-vertex (or per-batch) and Wasmoon's engine bootstrap is async. A new `WarpEvaluator` interface in `@yacad/kernel-manifold` decouples the kernel from Lua specifically; `@yacad/lua` provides the Wasmoon-backed implementation. The worker host pre-warms the evaluator at backend creation and passes it into `ManifoldKernel`. `params.code + params.values` is canonical-JSON-serializable and therefore hashable — `warp` participates in the cache the same way any other kernel node does.

**Tech Stack:** TypeScript strict mode; existing `wasmoon` runtime; existing `manifold-3d` WASM (specifically `Manifold.warp(func)`); vitest for unit/integration tests.

**Worktree:** `.worktrees/warp-lua` on branch `feat/warp-lua`. Branched from `main` at `9694cce`.

## Constraints / invariants

- **Determinism (CLAUDE.md #2):** Lua sandbox is no-I/O, no-clock, no-RNG; the Wasmoon engine in the warp path uses the same shared installer as the LuaNode runtime, configured with `random: { mode: 'disabled' }` (see Task 4). The warp's `params.code` and `params.values` are part of the semantic hash; the same `(code, values, child_mesh)` triple must produce byte-identical output.
- **Cache (CLAUDE.md #3):** `warp` produces a mesh; cache key is the existing `{ semantic_hash, produced_by }` shape; no schema change to the cache.
- **No Lua-runtime dep in kernel-manifold:** The `WarpEvaluator` interface lives in `@yacad/kernel-manifold`; the implementation in `@yacad/lua` imports the type. `kernel-manifold` does NOT depend on `@yacad/lua`. The worker host (which already depends on both) does the wiring.
- **Single-source-of-truth Lua sandbox.** LuaNode and warp must use the same install function. Divergence is configured, not duplicated. The shared installer (Task 4) is the only place library loading, dangerous-API stripping, RNG handling, and globals registration happens. This makes future security/determinism reviews tractable — every Wasmoon engine in the system goes through one path.
- **Purity contract for warp callbacks.** The Lua deformation function MUST be a pure function of `(x, y, z, params.values)` — no internal state, no side effects. Manifold iterates vertices in a deterministic order, so even a stateful callback would technically produce reproducible meshes today, but future LOD/preview tiers and any parallel evaluation depend on per-vertex purity. Random and clock are stripped from the sandbox so the obvious sources of non-determinism are unreachable; per-vertex statefulness via closures over locals is technically possible but contractually disallowed.

/**
 * Compiles a Lua deformation expression into a synchronous per-vertex callback
 * the kernel can hand to Manifold.warp(). The Lua engine is hidden behind the
 * interface so @yacad/kernel-manifold does not depend on a specific Lua runtime.
 *
 * The returned function receives the current (x, y, z) and returns the new
 * (x, y, z). It must be synchronous because Manifold's warp callback is sync.
 */
export interface WarpCallback {
  (x: number, y: number, z: number): readonly [number, number, number];
}

export interface WarpEvaluator {
  /**
   * Compile a Lua function body that mutates / returns a vertex position.
   * `values` is a table made available in the sandbox as `params` (parallel to
   * the LuaNode convention). The returned callback is reusable for every vertex
   * of one warp evaluation.
   */
  compile(code: string, values: Record<string, unknown>): Promise<WarpCallback>;
}
```

// packages/lua/src/sandbox.ts
import { LuaLibraries, type LuaEngine } from 'wasmoon';

/**
 * RNG policy for the sandbox.
 *
 *  - `disabled`: `math.random`, `math.randomseed`, and any string→function loader
 *    (`load`, `loadstring`, `string.dump`, etc.) are stripped. The kernel-side
 *    warp evaluator uses this — vertex-level randomness would break the cache.
 *  - `seeded`: `math.random` survives but is reseeded from caller-supplied
 *    `seedLo`/`seedHi` (two signed int64s, the shape Lua 5.4 `math.randomseed`
 *    expects). LuaNode uses this — authors can use random in DAG generation as
 *    long as the seed itself is a deterministic function of the definition + values.
 */
export type LuaRandomPolicy =
  | { readonly mode: 'disabled' }
  | { readonly mode: 'seeded'; readonly seedLo: bigint; readonly seedHi: bigint };

export interface LuaSandboxOptions {
  readonly random: LuaRandomPolicy;
  /** Globals installed on `_G` after the sandbox is set up. */
  readonly globals?: Readonly<Record<string, unknown>>;
}

/**
 * Single source of truth for the Lua sandbox. Every Wasmoon engine in the
 * system goes through this installer so the determinism + sandbox guarantees
 * (CLAUDE.md #2) hold uniformly across LuaNode and warp.
 *
 * Caller is responsible for creating the engine with `openStandardLibs: false`.
 */
export async function installLuaSandbox(
  engine: LuaEngine,
  options: LuaSandboxOptions,
): Promise<void> {
  // 1. Open only the pure stdlib chunks. Base brings pcall/error/type/tostring/
  //    tonumber/select/pairs/ipairs; we strip its dangerous entries below.
  //    Because the engine was created with openStandardLibs: false,
  //    os/io/package/coroutine/debug are never loaded at all.
  await engine.global.loadLibrary(LuaLibraries.Base);
  await engine.global.loadLibrary(LuaLibraries.Math);
  await engine.global.loadLibrary(LuaLibraries.String);
  await engine.global.loadLibrary(LuaLibraries.Table);

  // 2. RNG policy. Seed BEFORE stripping randomseed (the old plan had a bug
  //    here where the call to randomseed happened after randomseed had been
  //    nilled). For `disabled`, we still null both names defensively.
  if (options.random.mode === 'seeded') {
    await engine.doString(
      `math.randomseed(${options.random.seedLo.toString()}, ${options.random.seedHi.toString()})`,
    );
  }

  // 3. Strip impure / unwanted entries.
  const stripRandom = options.random.mode === 'disabled';
  await engine.doString(`
    math.randomseed = nil
    ${stripRandom ? 'math.random = nil' : ''}
    string.dump = nil
    dofile = nil
    loadfile = nil
    load = nil
    loadstring = nil
    require = nil
    print = nil
    collectgarbage = nil
  `);

  // 4. Install caller-supplied globals.
  if (options.globals) {
    for (const [name, value] of Object.entries(options.globals)) {
      engine.global.set(name, value);
    }
  }
}
```

```ts
// packages/lua/src/wasmoon-warp.ts
import { LuaFactory } from 'wasmoon';
import type { WarpEvaluator, WarpCallback } from '@yacad/kernel-manifold';
import { installLuaSandbox } from './sandbox';
import { LuaError } from './runtime';

export interface WasmoonWarpEvaluatorOptions {
  readonly wasmUrl?: string;
}

/**
 * Compiles Lua deformation functions to synchronous JS callbacks. A fresh
 * Wasmoon engine is created per compile — small overhead per warp node, but
 * keeps state clean (no cross-warp leakage). The returned callback is
 * synchronous: Manifold's warp() requires sync.
 *
 * Sandbox is configured with `random.mode = 'disabled'`: a vertex-deformation
 * function MUST be a pure function of (x, y, z, params). Random would make
 * the result vertex-order-dependent and break the purity contract.
 */
export class WasmoonWarpEvaluator implements WarpEvaluator {
  private readonly factory: LuaFactory;

  constructor(opts: WasmoonWarpEvaluatorOptions = {}) {
    this.factory = opts.wasmUrl ? new LuaFactory(opts.wasmUrl) : new LuaFactory();
  }

  async compile(code: string, values: Record<string, unknown>): Promise<WarpCallback> {
    const engine = await this.factory.createEngine({ openStandardLibs: false });
    await installLuaSandbox(engine, {
      random: { mode: 'disabled' },
      globals: { params: values },
    });
    // Wrap the user code into a function once. Each per-vertex call is then a
    // sync handle call (Wasmoon proxies Lua functions as sync JS functions).
    const wrapped = `function __yacad_warp(x, y, z)\n${code}\nend`;
    try {
      await engine.doString(wrapped);
    } catch (err) {
      engine.global.close();
      throw new LuaError(`warp compile error: ${(err as Error).message}`);
    }
    const fn = engine.global.get('__yacad_warp') as (
      x: number,
      y: number,
      z: number,
    ) => [number, number, number] | undefined;

    // Engine lifetime is tied to the returned callback. We don't expose close
    // here — the kernel discards the callback after one warp evaluation and the
    // engine goes out of scope.
    return (x, y, z) => {
      const out = fn(x, y, z);
      if (!Array.isArray(out) || out.length !== 3) {
        throw new Error(
          'warp Lua function must return three numbers: return new_x, new_y, new_z',
        );
      }
      return out as [number, number, number];
    };
  }
}
```

```ts
private async evaluateWarp(
  node: Node,
  childGeometries: readonly Geometry[],
): Promise<KernelResult> {
  if (!this.options.warpEvaluator) {
    throw new KernelError(
      `node ${node.id}: warp requires a WarpEvaluator (none supplied to the kernel)`,
    );
  }
  const mesh = asMesh(childGeometries[0]!, node.id, 0);
  const code = node.params['code'] as string;
  const values = (node.params['values'] as Record<string, unknown>) ?? {};

  const compileStart = performance.now();
  const cb = await this.options.warpEvaluator.compile(code, values);
  const compileMs = performance.now() - compileStart;

  const importStart = performance.now();
  const solid = this.toSolid(mesh);
  const importMs = performance.now() - importStart;

  const opStart = performance.now();
  let result: Solid;
  try {
    result = solid.warp((v) => {
      const [nx, ny, nz] = cb(v[0], v[1], v[2]);
      v[0] = nx;
      v[1] = ny;
      v[2] = nz;
    });
  } finally {
    solid.delete();
  }
  const opMs = performance.now() - opStart;

  const exportStart = performance.now();
  try {
    const out = this.toMesh(result);
    return {
      geometry: { kind: '3d', mesh: out },
      // Charge Lua compile to importMs since it's setup, not the core op.
      timings: { importMs: importMs + compileMs, opMs, exportMs: performance.now() - exportStart },
    };
  } finally {
    result.delete();
  }
}
```

- [ ] **Step 1: Write the Lua definition**

`TORUS_KNOT_DEFINITION` is a Lua-node definition with schema params:
- `p` (int, default 1, min 1, max 8): thread passes through the donut hole.
- `q` (int, default 3, min 1, max 8): thread circles the donut.
- `majorRadius` (number, default 25)
- `minorRadius` (number, default 10)
- `threadRadius` (number, default 3.75)
- `circularSegments` (int, default 24, min 3, max 64)

Output type `'3d'`, no inputs.

Lua body (port of the Matlab Knot example):

```lua
local function gcd(a, b)
  while b ~= 0 do a, b = b, a % b end
  return a
end

local p = params.p
local q = params.q
local kLoops = gcd(p, q)
local pk = p / kLoops
local qk = q / kLoops
local n = params.circularSegments
local m = math.max(8, math.floor(n * qk * params.majorRadius / params.threadRadius))
local offset = 2

-- Cross-section: a small circle at x = offset (the radial distance from the
-- revolve axis that becomes the unwarped torus minor circle).
local circle = geo.translate_2d(
  { offset = { offset, 0 } },
  { geo.circle({ radius = 1, segments = n }) }
)
local revolved = geo.revolve({ segments = m }, { circle })

-- Warp shapes the revolved torus into the (p, q) knot.
local code = [[
  local pk = params.pk
  local qk = params.qk
  local majorRadius = params.majorRadius
  local minorRadius = params.minorRadius
  local threadRadius = params.threadRadius
  local offset = params.offset

  local psi = qk * math.atan(x, y)  -- math.atan in Lua 5.3+ takes (y, x); we pass (x, y) to match the JS atan2(v[0], v[1])
  local theta = psi * pk / qk
  local x1 = math.sqrt(x * x + y * y)
  local phi = math.atan(x1 - offset, z)
  local px = threadRadius * math.cos(phi)
  local pz = threadRadius * math.sin(phi)
  -- rotate (0, pz) by -atan(pk*minor, qk*r) where r = majorRadius + minor*cos(theta)
  local r = majorRadius + minorRadius * math.cos(theta)
  local a1 = -math.atan(pk * minorRadius, qk * r)
  local c, s = math.cos(a1), math.sin(a1)
  local py_new = 0 * c - pz * s
  local pz_new = 0 * s + pz * c
  -- accumulate
  local x2 = px + minorRadius
  local y2 = py_new
  local z2 = pz_new
  -- rotate (x2, z2) by -theta
  local a2 = -theta
  local c2, s2 = math.cos(a2), math.sin(a2)
  local x3 = x2 * c2 - z2 * s2
  local z3 = x2 * s2 + z2 * c2
  -- rotate (x3 + majorRadius, y2) by psi
  local x4 = x3 + majorRadius
  local y4 = y2
  local c3, s3 = math.cos(psi), math.sin(psi)
  return x4 * c3 - y4 * s3, x4 * s3 + y4 * c3, z3
]]

return geo.warp(
  { code = code, values = {
    pk = pk, qk = qk,
    majorRadius = params.majorRadius,
    minorRadius = params.minorRadius,
    threadRadius = params.threadRadius,
    offset = offset,
  } },
  { revolved }
)
```

(NOTE: Lua 5.3's `math.atan(y, x)` is two-arg atan2. Double-check argument order against Wasmoon's bundled Lua version when porting from JS `Math.atan2(a, b)` → JS docs say atan2(y, x). The original JS uses `atan2(v[0], v[1])` which is `atan2(x_old, y_old)`. Lua becomes `math.atan(x_old, y_old)`. Verify with a unit test.)

## Risk register

- **Wasmoon engine lifetime.** The warp callback holds a reference to a Lua engine; the engine must outlive every per-vertex call. We create one engine per warp eval; if Manifold's warp() is synchronous (it is) this is fine. If we later batch via `warpBatch`, same story.
- **Lua function return shape.** Wasmoon may return a `multret` as a JS array or a single scalar depending on how it's wrapped. Test verifies the three-element-array path; the implementation throws a clear error otherwise.
- **`math.atan` argument order.** The JS atan2 in the original source uses `(v[0], v[1])` and `(x1 - offset, v[2])` — port carefully and verify with a small unit test on the warp callback itself (separate from Manifold).
- **Performance.** Per-vertex Wasmoon call is ~µs-scale. A 24×100 torus knot has 2400 vertices → ~few ms. If we ship something larger, revisit using `warpBatch`.
- **Cache invalidation.** `params.values` JSON-canonicalization must be stable. The existing canonical infra already covers nested objects, so this should be free — but add a hash test on `warp` that asserts equal values produce equal hashes.
