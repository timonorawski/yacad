# 2D Layer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Phase 2 of the parametric CAD roadmap — 14 new node types + extensions to 3 existing ones — turning the platform from "MVP 3D object assembly" into "MVP CAD." Spec: [docs/superpowers/specs/2026-05-27-2d-layer-design.md](../specs/2026-05-27-2d-layer-design.md).

**Architecture:** Extend the existing dual-type system with a `Geometry` discriminated union flowing through kernel and engine. Cache stores by artifact kind (mesh vs crossSection). `KernelNodeType.output` widened to allow a per-instance resolver so type-overloaded ops (`union`, `difference`, `intersection`, `hull`) work for both 2D and 3D without duplication. LuaNode's `schema.output: '2d'` becomes a first-class composition — zero code changes in `@yacad/lua`, but new examples and tests prove it.

**Tech Stack:** TypeScript / pnpm workspaces, Vitest, Manifold 3.5.0 (CrossSection, Extrude, Revolve, Offset, Hull, Refine native), three.js (rendering), Wasmoon (Lua, unchanged).

**Spec reference:** Read [docs/superpowers/specs/2026-05-27-2d-layer-design.md](../specs/2026-05-27-2d-layer-design.md) end-to-end before starting. If you find tension between the plan and the spec, the spec wins — pause and raise it rather than improvise.

**Project conventions (skim before starting):**

- Tests live next to source: `foo.ts` + `foo.test.ts` colocated. Vitest, no `__tests__/` subdirs.
- `pnpm --filter @yacad/<pkg> test` runs one package's tests; `pnpm vitest run packages/<pkg>` works too.
- `pnpm build` is the type-correctness gate (`tsc -b` with project references).
- Vitest aliases `@yacad/*` to source via `vitest.config.ts` — no `pnpm build` needed between test runs.
- Existing validation helpers in `packages/dag/src/validate.ts` (`asRecord`, `num`, `posNum`, `vec3`, `optBool`, `optSegments`) — REUSE for new validators, do not duplicate.
- `NOOP_RESOLVER` and `registerNodeType`/`unregisterNodeType` are exported from `@yacad/dag` (Phase 1 additions). Use them for test isolation around dynamically-registered types.
- Commit cadence: commit after every passing test transition. Conventional-commit prefixes (`feat:`, `refactor:`, `test:`, `chore:`).
- TDD: every new behavior arrives with a failing test first. No exceptions.
- Run `pnpm format` and `pnpm lint` before each chunk-level commit.

**Phase 1 foundation in place:**

- `KernelNodeType | ExpandableNodeType` discriminated union (`packages/dag/src/registry.ts`)
- `ExpandableNodeType.inputNames(params, resolver)` for `__input_ref` by-name resolution
- `DefinitionResolver` generic interface; `NOOP_RESOLVER` exported
- Engine constructor uses options bag: `new Engine(store, kernel, { resolver?, engineVersion? })`
- `EvaluateResult.geometry` will REPLACE `EvaluateResult.mesh` in this phase — currently `.mesh: Mesh`. All callers update.
- Cache `Artifact = MeshArtifact | BBoxArtifact | LuaDefinitionArtifact`; we add a fourth.
- `NodeEval` carries `importMs`, `opMs`, `exportMs` from `kernel.evaluateTimed`. Both 2D and 3D operations populate these.

**Reserved-prefix invariant:** `__`-prefixed node types are engine-internal (`__input_ref`). `buildGraph` rejects any authored document containing them. The new node types in this phase use plain names (no `__`).

```ts
import type { Vec2 } from '@yacad/geometry';

export function vec2(p: Record<string, unknown>, key: string, path: string): Vec2 {
  const v = p[key];
  if (!Array.isArray(v) || v.length !== 2) {
    throw new DagError(`"${key}" must be a 2-element array`, path);
  }
  for (let i = 0; i < 2; i++) {
    const n = v[i];
    if (typeof n !== 'number' || !Number.isFinite(n)) {
      throw new DagError(`"${key}[${i}]" must be a finite number`, path);
    }
  }
  return [v[0] as number, v[1] as number];
}

export function posVec2(p: Record<string, unknown>, key: string, path: string): Vec2 {
  const v = vec2(p, key, path);
  for (let i = 0; i < 2; i++) {
    if (v[i]! <= 0) throw new DagError(`"${key}[${i}]" must be greater than 0`, path);
  }
  return v;
}
```

## Chunk 2: Kernel signature pivot + first 2D primitive (`circle`)

The riskiest chunk in this phase. `Kernel.evaluate`/`evaluateTimed` signature shifts from `(node, Mesh[]) → Mesh` to `(node, Geometry[]) → Geometry`. All seven existing kernel handlers are wrapped to return `{ kind: '3d', mesh }`. Engine's `walk()` updates accordingly. The new `circle` node type validates the end-to-end pipeline for 2D.

`EvaluateResult.mesh` is REPLACED by `EvaluateResult.geometry: Geometry`. Callers update.

**Acceptance:** `pnpm test && pnpm build && pnpm lint && pnpm format:check` green. Studio still renders 3D scenes (rendering 2D is chunk 6 — for now, attempting to render a 2D root throws a clear error).

### Chunk-2 constraints (read before starting)

**META-INSTRUCTION:** Where these constraints conflict with code blocks below, the constraints win. The kernel signature pivot affects many files; the task code blocks show the changes incrementally, but if any specific signature looks wrong in a block, refer back here.

1. **Kernel signature:** `evaluateTimed(node, childGeometries: readonly Geometry[]): KernelResult` where `KernelResult = { geometry: Geometry; timings: { importMs: number; opMs: number; exportMs: number } }`. The `geometry` field replaces the old `mesh` field. `evaluate(...)` (non-timed convenience wrapper) returns `Geometry`.

2. **Engine.walk:** child results from recursive `walk()` calls are now `Geometry[]`. Cache lookup uses `artifactKindFor(node.outputType)` to pick between `'mesh'` and `'crossSection'`. Cache write uses the same. The kernel branch passes `Geometry[]` to `kernel.evaluateTimed`, receives a `Geometry`, stores it under the correct artifact kind.

3. **EvaluateResult.geometry replaces .mesh.** Existing 12+ callers (in `packages/worker/src/host.ts`, `packages/e2e/src/pipeline.ts`, `apps/studio/src/App.svelte`, `bench/*`) update to read `result.geometry` and pattern-match on `.kind`. The worker protocol's `EvaluateOk.mesh` field becomes `EvaluateOk.geometry`. Studio's render path stays 3D-only for now: if it receives a `{ kind: '2d' }` result, it throws an explicit "2D rendering not yet implemented" error (lifted in chunk 6).

4. **All seven existing handlers wrap their output:** Inside the kernel switch, every existing handler ends with `return { mesh: <built-mesh>, timings: {...} }` today. They become `return { geometry: { kind: '3d', mesh }, timings: {...} }`. Behaviorally identical for 3D consumers.

5. **`circle` is the validation primitive.** Simplest 2D node: `{ params: { radius, segments? }, output: '2d', no children }`. Implementation uses Manifold's `CrossSection.circle(radius, segments)` (or equivalent — verify in Manifold's encapsulated-types.d.ts at install time; the static constructor name may be `CrossSection.makeCircle` or constructed via polygon points instead).

6. **`artifactKindFor` is a small helper** in `packages/engine/src/engine.ts`:
   ```ts
   function artifactKindFor(geometryType: GeometryType): ArtifactKind {
     return geometryType === '2d' ? 'crossSection' : 'mesh';
   }
   ```

```ts
): Vec2[] {

  if (!Array.isArray(v)) {
    throw new DagError(`"${key}" must be an array`, path);
  }
  if (v.length < minLen) {
    throw new DagError(`"${key}" must have at least ${minLen} entries`, path);
  }
  return v.map((entry, i) => {
    if (!Array.isArray(entry) || entry.length !== 2) {
      throw new DagError(`"${key}[${i}]" must be a 2-element array`, path);
    }
    for (let j = 0; j < 2; j++) {
      const n = entry[j];
      if (typeof n !== 'number' || !Number.isFinite(n)) {
        throw new DagError(`"${key}[${i}][${j}]" must be a finite number`, path);
      }
    }
    return [entry[0] as number, entry[1] as number] as Vec2;
  });
}
```

```ts
import type { Vec2 } from '@yacad/geometry';

/**
 * Closed Catmull-Rom interpolation. Given N control points and S segments per
 * curve, returns N*S output points along the curve. The curve passes through
 * every control point (interpolating, not approximating).
 *
 * Tension 0.5 is the "standard" Catmull-Rom. Lower values make tighter curves.
 *
 * Pure function — same inputs always produce same outputs (cache-key safe).
 */
export function catmullRomClosed(
  points: ReadonlyArray<readonly [number, number]>,
  segmentsPerCurve: number,
  tension: number,
): Vec2[] {
  const n = points.length;
  const out: Vec2[] = [];
  const t = 1 - tension; // standard formulation uses `t` as the inverse
  for (let i = 0; i < n; i++) {
    const p0 = points[(i - 1 + n) % n]!;
    const p1 = points[i]!;
    const p2 = points[(i + 1) % n]!;
    const p3 = points[(i + 2) % n]!;
    for (let s = 0; s < segmentsPerCurve; s++) {
      const u = s / segmentsPerCurve;
      const u2 = u * u;
      const u3 = u2 * u;
      // Catmull-Rom basis (matrix form with tension)
      const b0 = -t * u3 + 2 * t * u2 - t * u;
      const b1 = (2 - t) * u3 + (t - 3) * u2 + 1;
      const b2 = (t - 2) * u3 + (3 - 2 * t) * u2 + t * u;
      const b3 = t * u3 - t * u2;
      const x = b0 * p0[0] + b1 * p1[0] + b2 * p2[0] + b3 * p3[0];
      const y = b0 * p0[1] + b1 * p1[1] + b2 * p2[1] + b3 * p3[1];
      out.push([x, y]);
    }
  }
  return out;
}
```

```ts
/** An N-ary operation that accepts ≥minChildren children, all of the same
 *  output type (either all-2D or all-3D). Output type matches the children's
 *  type. No params. Used by union, difference, intersection, hull. */
function overloaded(type: string, minChildren: number): KernelNodeType {
  return {
    kind: 'kernel',
    type,
    output: (children) => children[0]!.outputType,
    checkChildren(children, path) {
      if (children.length < minChildren) {
        throw new DagError(
          `"${type}" requires at least ${minChildren} child${minChildren > 1 ? 'ren' : ''}`,
          path,
        );
      }
      const first = children[0]!.outputType;
      for (let i = 1; i < children.length; i++) {
        if (children[i]!.outputType !== first) {
          throw new DagError(
            `"${type}" expects all children of the same dimension; ` +
              `got ${children.map((c) => c.outputType).join(', ')}`,
            path,
          );
        }
      }
    },
    normalizeParams(params, path) {
      asRecord(params, path);
      return {};
    },
  };
}
```

```ts
import * as THREE from 'three';
import type { CrossSection } from '@yacad/geometry';
import type ManifoldModule from 'manifold-3d';

/** Build a flat (z=0) BufferGeometry from a CrossSection's polygons.
 *  Triangulation uses Manifold's triangulate helper. */
export function crossSectionToBufferGeometry(
  cs: CrossSection,
  api: typeof ManifoldModule,
): THREE.BufferGeometry {
  const polygons = cs.polygons as ReadonlyArray<ReadonlyArray<[number, number]>>;
  // Manifold's triangulate returns a flat Uint32Array of triangle indices.
  // Verify the API — it may be `Manifold.triangulate(polygons)` or via a
  // module-level export. Adjust accordingly.
  const tris = api.triangulate(polygons);
  // Flatten polygons to a single vertex list (x, y, 0 per vertex).
  const verts: number[] = [];
  for (const poly of polygons) {
    for (const [x, y] of poly) {
      verts.push(x, y, 0);
    }
  }
  const geom = new THREE.BufferGeometry();
  geom.setAttribute('position', new THREE.Float32BufferAttribute(verts, 3));
  geom.setIndex(new THREE.Uint32BufferAttribute(tris, 1));
  geom.computeVertexNormals();
  return geom;
}
```

- [ ] **Step 2: Add `geometryToObject3D` dispatch in `geometry.ts`**

```ts
import type { Geometry } from '@yacad/geometry';
import { crossSectionToBufferGeometry } from './cross-section-mesh';

export function geometryToObject3D(geometry: Geometry, api: typeof ManifoldModule): THREE.Object3D {
  if (geometry.kind === '3d') {
    const buf = meshToBufferGeometry(geometry.mesh);
    return new THREE.Mesh(buf, defaultMaterial());
  }
  // 2D: fill + outline group
  const buf = crossSectionToBufferGeometry(geometry.section, api);
  const fill = new THREE.Mesh(buf, defaultFill2dMaterial());
  const outline = buildOutline(geometry.section);
  const group = new THREE.Group();
  group.add(fill);
  group.add(outline);
  return group;
}

function defaultFill2dMaterial(): THREE.Material {
  return new THREE.MeshBasicMaterial({
    color: 0x88aacc,
    transparent: true,
    opacity: 0.4,
    side: THREE.DoubleSide,
  });
}

function buildOutline(cs: CrossSection): THREE.Object3D {
  const group = new THREE.Group();
  for (const polygon of cs.polygons) {
    const positions = new Float32Array(polygon.length * 3 + 3);
    for (let i = 0; i < polygon.length; i++) {
      positions[i * 3] = polygon[i]![0];
      positions[i * 3 + 1] = polygon[i]![1];
      positions[i * 3 + 2] = 0;
    }
    // Close the loop
    positions[polygon.length * 3] = polygon[0]![0];
    positions[polygon.length * 3 + 1] = polygon[0]![1];
    positions[polygon.length * 3 + 2] = 0;
    const geom = new THREE.BufferGeometry();
    geom.setAttribute('position', new THREE.BufferAttribute(positions, 3));
    const line = new THREE.Line(geom, new THREE.LineBasicMaterial({ color: 0x000000 }));
    group.add(line);
  }
  return group;
}
```
